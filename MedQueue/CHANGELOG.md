# Changelog

## [1.0.0] — 2026-03-03 🚀 Первый стабильный релиз

> **Масштабное обновление.** MedQueue выходит из стадии беты и получает первый полноценный стабильный релиз.  
> Платформа полностью готова к демонстрации и реальному использованию.

### Что нового в 1.0.0

#### Безопасность и доступ

- **Ролевая маршрутизация** — `enforceRoleAccess()` в `script.js` защищает все страницы по роли:
  - Пациент не может попасть на `doctor.html` или `admin-panel.html`
  - Врач и администратор не могут войти в экосистему пациента
  - При попытке неверного входа — автоматический редирект на нужную страницу
- **ngrok-совместимость** — все `fetch()`-запросы во всех HTML-файлах теперь содержат заголовок `ngrok-skip-browser-warning: true`; сайт полноценно работает через ngrok-туннель без перебоев
- Django `settings.py` — `ALLOWED_HOSTS` и `CSRF_TRUSTED_ORIGINS` включают ngrok-домен

#### Страница врача (`doctor.html`)

- **ИИ-ассистент МедAI** — полностью рабочая панель: кнопки получили иконку `🤖`, заголовок панели — `🤖 МедAI — помощник`, кнопка отправки `➤`, кнопка закрытия `✕`
- `apiFetch()` централизует все заголовки включая `ngrok-skip-browser-warning`
- Мобильный вид: топбар скрывает имя/аватар, таблица записей оборачивается в горизонтально прокручиваемый контейнер (`overflow-x: auto`, `min-width: 560px`)

#### Страница регистрации/записи (`recording.html`)

- **Удалён AI-чат виджет** — весь блок (стили + HTML + JS) удалён, страница стала чище
- Мобильный вид: специальности уменьшены до `font-size: 11px`, `form-grid-2` → одна колонка, `dt-picker` — вертикальный стек, `clinics-grid` → одна колонка, декоративные эмоджи скрыты

#### Личный кабинет (`profile.html`)

- Исправлено переполнение блоков на мобильных устройствах — добавлены правила `overflow-guard`
- Заголовок записи (`appt-header`) перестраивается в колонку на узких экранах
- Кнопки действий записи — полная ширина на мобильных
- Все 5 `fetch()`-вызовов API включают `ngrok-skip-browser-warning: true`

#### Страница авторизации (`auth.html`)

- Мобильный вид: `padding: 22px 18px`, кнопки ролей сжаты, переключатель темы смещён в правый нижний угол
- Все 8 `fetch()`-вызовов (login, register, verify, resend, password-reset, validate-doctor-code) включают `ngrok-skip-browser-warning`

#### Глобальные стили и JS (`styles.css`, `script.js`)

- `overflow-x: hidden` на `body` — устранено горизонтальное переполнение на мобильных
- `getAuthHeaders()` и `ensureFreshToken()` в `script.js` включают `ngrok-skip-browser-warning`
- Переключатель темы — правый нижний угол на экранах ≤ 480px (не перекрывает контент)
- AI-чат: `width: min(340px, calc(100vw - 32px))` — не выходит за экран на мобильных
- Кнопка отправки `.submit-wrap .btn` — 100% ширины на мобильных

#### Версия сайта

- Во всех страницах (`main.html`, `recording.html`, `profile.html`, `status.html`, `contacts and about.html`) — **Версия 3.0**

- Удалены зависимости `leaflet@1.9.4` (CSS + JS) — использовалось через CDN

  - `new mapgl.Popup(...)` вместо `L.marker.bindPopup(...)` — попап открывается по клику с авто-закрытием предыдущего
  - Координаты передаются в формате `[lon, lat]` (требование 2ГИС API); автоматическая перестановка из `[lat, lon]`
- CSS-правила для `.leaflet-popup-content-wrapper`, `.leaflet-popup-tip`, `.leaflet-pane` заменены правилами для `.mapgl-popup-container`
- Сохранен: попап с названием, типом, адресом, размером очереди и ссылкой «Подробнее»

#### Добавлено
- **JWT-аутентификация** (`djangorestframework-simplejwt==5.3.1`)

  - `POST /api/auth/verify/` (подтверждение email) тоже возвращает токены — пользователь сразу авторизован после регистрации
